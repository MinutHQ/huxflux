## Connecting to a server

### Local server (automatic)

If you installed the server and desktop app together (via the installer or setup wizard), the desktop app auto-connects on first launch. No manual configuration needed.

### Remote server

To connect to a server on another machine, get its connection string:

```sh
huxflux status    # prints URL, token, and QR code
```

Then paste the `huxflux://...` string into the server switcher in the sidebar, or go to **Settings > Servers** and add it manually.

The token authenticates the app to your server. Treat it like a password.

## Models and providers

Huxflux supports multiple AI providers (Claude Code, Codex, OpenCode). Configure the default provider and model under **Settings > Models**.

- Pick a **provider** first (only installed CLIs are shown)
- Then pick a **model** from that provider
- These defaults apply to new agents. You can override per-agent in the model selector at the bottom of the chat input.

| Model | When to use |
|---|---|
| **Haiku 4.5** | Fast and cheap. Good for small, well-defined tasks. |
| **Sonnet 4.6** | Balanced. Best default for most work. |
| **Opus 4.6** | Most capable. Use for complex reasoning or architecture decisions. |

## Repositories

Add repositories under **Settings > Git**. Enter the local path to a git repository, or use "Add folder" for non-git directories.

Each agent gets an isolated git worktree under `~/huxflux/workspaces/`. Your main checkout is untouched.

<Callout type="warn">
  Only add repositories you trust. Malicious content in a repository (README,
  comments, test fixtures) can instruct Claude to run arbitrary commands.
</Callout>

## Network

By default, `huxflux` listens on port **4321**. Override with the `PORT` environment variable:

```sh
PORT=3002 huxflux start
```

We recommend keeping this port off the public internet and accessing it via [Tailscale](https://tailscale.com) instead. See the [security guide](/docs/guides/security) for details.

## Updates

### Desktop app

The desktop app checks for updates automatically. Configure auto-update behavior under **Settings > Updates**.

### Server

The server checks for new versions every 6 hours. You can:
- Update from the desktop app under **Settings > Updates**
- Update from the CLI: `huxflux update`
- Enable auto-updates: `huxflux config auto-update on`

When auto-update is enabled, the server updates itself when no agents are running.
