## One-line install

The fastest way to get started. Installs the server, sets up your environment, and optionally installs the desktop app.

```sh
curl -fsSL https://raw.githubusercontent.com/MinutHQ/huxflux/main/install.sh | bash
```

The installer will:
1. Check that Node.js 22.6+ is installed
2. Ask which release channel (stable or beta)
3. Install the `huxflux` CLI via npm
4. Fix your PATH if needed (asks permission)
5. Launch the setup wizard

To install the beta channel directly without the prompt:

```sh
curl -fsSL https://raw.githubusercontent.com/MinutHQ/huxflux/main/install.sh | bash -s -- --beta
```

Switch channels later with `huxflux config channel beta` or `huxflux config channel stable`.

### Setup wizard

The wizard detects your environment and walks you through:

- **Server** -- creates the config, database, and starts the API
- **Auto-start** -- optionally installs a system service (LaunchAgent on macOS, systemd on Linux) so Huxflux starts on login
- **Desktop app** -- downloads and installs the native app (DMG on macOS, AppImage on Linux)

The desktop app auto-connects to your local server on first launch.

### Uninstall

```sh
curl -fsSL https://raw.githubusercontent.com/MinutHQ/huxflux/main/install.sh | bash -s -- --uninstall
```

This removes the server, desktop app, system service, and all data.

## Desktop app

Download the desktop app directly if you already have a server running.

[Download on GitHub](https://github.com/MinutHQ/huxflux/releases/latest)

Available as `.dmg` (macOS), `.AppImage` (Linux), and `.deb` (Linux).

**macOS -- app is unsigned**

If macOS blocks the app on first launch, remove the quarantine flag:

```sh
xattr -d com.apple.quarantine /Applications/Huxflux.app
```

**Linux -- FUSE required for AppImage**

If you see an error about `libfuse.so.2`:

```sh
sudo apt install libfuse2
```

## Manual server install

If you prefer to install the server manually without the setup wizard:

### Requirements

- **Node.js** 22.6 or later

<Steps>

<Step>

### Install globally

```sh
npm install -g @minuthq/huxflux
```

</Step>

<Step>

### Run the setup wizard

```sh
huxflux setup
```

</Step>

<Step>

### Or start manually

```sh
huxflux start
huxflux open    # opens web app and auto-connects
```

</Step>

</Steps>

## HTTPS / Remote access

When using the hosted web app (served over HTTPS) with a self-hosted server, browsers block plain `ws://` connections. Put your server behind HTTPS using a reverse proxy.

### Tailscale (recommended)

[Tailscale](https://tailscale.com) encrypts all traffic with WireGuard. Install it on both machines, then connect using your Tailscale IP:

```
huxflux://100.x.x.x:4321?token=...
```

### nginx + Let's Encrypt

```sh
sudo apt install nginx certbot python3-certbot-nginx
sudo certbot --nginx -d yourdomain.com
```

Add this to your nginx site config:

```nginx
location / {
    proxy_pass http://localhost:4321;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
    proxy_set_header Host $host;
}
```

The `Upgrade` headers are required for WebSocket proxying.

## Troubleshooting

**`huxflux` not found after install**

The installer offers to fix this automatically. If you skipped it, add npm's global bin directory to your PATH:

```sh
export PATH="$(npm config get prefix)/bin:$PATH"
```

Add this to `~/.bashrc` or `~/.zshrc` and run `source ~/.zshrc`.

**Desktop app doesn't connect to local server**

Make sure the server is running (`huxflux status`). The desktop app reads `~/huxflux/connection.json` on launch. If the server wasn't running when the app started, restart the app.
